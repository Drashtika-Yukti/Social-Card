# 🎨 Drashtika SocialForge: High-Performance Social Card Generator

**Drashtika SocialForge** is a professional, secure, and production-ready Go microservice designed to instantly transform Markdown content into beautiful, high-resolution social media preview images (OpenGraph cards).

![Drashtika SocialForge Showcase](https://via.placeholder.com/800x420.png?text=Drashtika+SocialForge+Showcase)

---

### 🚀 The Problem & Objective
**The Current Issue:** Creating consistent social sharing images manually for blog posts, products, or marketing campaigns is slow, expensive, and error-prone. 

**The Objective:** Provide a rock-solid API that developers can integrate into their CMS or backend to generate hundreds of pixel-perfect social cards automatically every second.

---

### 🔥 Key Features
- **Headless Chrome Engine**: Uses `chromedp` to render native-browser snapshots for 100% visual accuracy.
- **Intelligent In-Memory Caching**: Second-request latency is near-zero (approx. 2ms) thanks to internal content-hashing.
- **Dynamic Theming**: Swap between `Light` and `Dark` modes and customize brand `Accent Colors` in real-time.
- **Structured Style Overrides**: Securely override padding, font sizes, and background colors via JSON.
- **Production Security**:
  - `X-API-KEY` Authentication.
  - Native HTML Sanitization (Prevents XSS attacks).
  - Built-in Rate Limiting.
- **Iron-Box Boundaries**: Guaranteed layout stability with line-clamping and overflow protection.

---

### 📂 Project Structure
Built with a modular, scalable architecture:
- `cmd/server/`: The entry point for the service.
- `internal/api/`: HTTP Handlers, Routing, and Security Middleware.
- `internal/generator/`: Core Markdown parsing and Chromium screenshot logic.
- `internal/templates/`: The premium CSS/HTML glassmorphism design.
- `internal/config/`: Environment-based settings management.

### 🛠️ Getting Started

#### Option A: Docker (Recommended)
You don't need Go or Chrome installed. Just run:
```bash
docker-compose up --build
```
The service will be live at `http://localhost:7860`.

#### Option B: Local Run
```bash
go mod download
$env:API_KEY="your-secret-key"
go run cmd/server/main.go
```

### 📡 API Example
**POST** `/generate`
**Headers:** `X-API-KEY: your-secret-key`

```json
{
  "title": "Welcome to Drashtika",
  "author": "Engineering Team",
  "theme": "dark",
  "accent_color": "#38bdf8",
  "markdown": "## Success!\n\nThis card was generated by the **SocialForge** engine.",
  "style_overrides": {
      "padding_px": 50,
      "title_font_size_px": 20
  }
}
```

### 🚀 Deployment (CD)

This project is configured for **Continuous Deployment**. Every push to `main` automatically publishes a new Docker image to the **GitHub Container Registry (GHCR)**.

#### Deploy to Google Cloud Run (Recommended)
Once your image is published to GHCR, you can deploy it to a live URL using the following command (requires [Google Cloud SDK](https://cloud.google.com/sdk)):

```bash
gcloud run deploy social-forge \
  --image ghcr.io/drashtika-yukti/social-card:latest \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated \
  --set-env-vars="API_KEY=your-secret-key,ENV=production"
```
